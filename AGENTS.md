# Codex 作業ルール

このリポジトリは、ユーザーがCodexへ依頼した技術記事をGitHub Pagesへ公開するためのプロジェクトです。

## 記事公開の自動ワークフロー

ユーザーが「記事を作成して」「記事にまとめて」「Skill Notesへ追加して」など、記事作成を明示的に依頼した場合、その依頼には次の作業まで含まれるものとして扱ってください。

1. 質問内容を調査する
2. 公式ドキュメントや一次資料を優先して事実を確認する
3. `.github/ARTICLE_TEMPLATE.md`と既存記事を参考に記事を作成する
4. `src/content/articles/{slug}/index.md`へ配置する
5. Frontmatterの`draft`を`false`にして公開対象にする
6. `npm run format`を実行する
7. `npm run validate`を実行する
8. 記事作成に必要なファイルだけをコミットする
9. 現在の公開ブランチを`origin`へpushする
10. GitHub Actionsの`Deploy to GitHub Pages`が成功するまで確認する
11. 公開記事のURLへアクセスできることを確認する
12. 最終回答で記事URLを伝える

ユーザーへ、手動での`git add`、`git commit`、`git push`を依頼しないでください。必要な認証や実行許可を取得できる場合は、Codex自身が完了させてください。

## 公開URL

既定の公開URLは次のとおりです。

```text
https://kajinagi0001-bit.github.io/Skill-notes/
```

記事URLは次の形式です。

```text
https://kajinagi0001-bit.github.io/Skill-notes/articles/{slug}/
```

`SITE_URL`や`BASE_PATH`が変更されている場合は、現在の設定とGitHub Pagesのデプロイ結果を優先してください。

## 記事の品質基準

- 日本語で執筆する
- slugには英小文字、数字、ハイフンだけを使う
- 対象バージョン、検証日、前提条件を可能な限り明記する
- 公式ドキュメント、仕様、論文などの一次資料を優先する
- 事実、推測、推奨を区別する
- コマンドやコードは、可能な範囲で実際に検証する
- 参考資料へのリンクを記事末尾へ掲載する
- 秘密情報、個人情報、認証情報を記事やコミットへ含めない
- 既存記事と内容が重複する場合は、新規記事ではなく既存記事の更新も検討する

## Gitの扱い

- 作業開始時に`git status`と現在のブランチを確認する
- ユーザーの未コミット変更を上書き、削除、または無断でコミットしない
- 記事と、その公開に直接必要な変更だけをステージする
- コミットメッセージは記事の内容が分かる英語にする
- push前に`npm run validate`が成功していることを必須とする
- 検証に失敗した場合はpushせず、原因を修正して再検証する
- pushが拒否された場合は、リモートの変更を確認して安全に統合してから再試行する
- 強制pushは、ユーザーから明示的に指示されない限り行わない

## デプロイ確認

push後は、GitHub Actionsのデプロイを確認してください。

- GitHub CLIが利用可能なら、対象コミットのActions実行を監視する
- GitHub CLIが利用できなければ、GitHubのActionsまたは公開URLを確認する
- デプロイ失敗時はログを調べ、修正、再検証、再pushまで行う
- 公開URLで新しい記事が確認できるまで、作業完了とはしない

GitHub Pagesの反映には少し時間がかかる場合があります。反映待ちだけを理由に、ユーザーへ手作業を戻さないでください。

## 自動公開しない場合

次の場合は、記事の作成やpushを自動実行しません。

- ユーザーが単に技術的な質問をしているだけの場合
- ユーザーが下書き、レビュー、案だけを求めている場合
- ユーザーがpushや公開をしないよう明示した場合
- 公開すると秘密情報、個人情報、ライセンス違反などの危険がある場合
- GitHubの認証や権限がなく、許可を得てもpushできない場合

ユーザーが`draft: true`を明示した場合は、記事の作成と検証まで行い、公開用pushは行わないでください。

## 最終報告

記事を公開した後の最終回答には、最低限次を含めてください。

- 記事タイトル
- 公開URL
- `npm run validate`の結果
- GitHub Pagesデプロイの結果

ユーザーは、最終回答に記載されたURLを開くか再読み込みするだけで記事を閲覧できる状態にしてください。
