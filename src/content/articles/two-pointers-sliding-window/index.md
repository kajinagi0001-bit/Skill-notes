**尺取法**は、配列や文字列の連続した区間を、左右二つのポインタを動かしながら効率よく調べるアルゴリズムです。英語では **two pointers** や **sliding window** と呼ばれます。

すべての区間を一つずつ調べると、多くの場合は`O(N²)`の時間がかかります。区間の端を後戻りさせずに処理できる問題では、尺取法を使うことで`O(N)`まで改善できます。

## 尺取法が扱うもの

長さ`N`の配列に対して、半開区間`[left, right)`を考えます。

```text
添字:    0   1   2   3   4   5
配列:   [2,  1,  3,  2,  4,  1]
             └───────┘
             left    right

調べている区間は left 以上、right 未満
```

`left`は区間の先頭、`right`は区間の末尾の一つ後ろを表します。半開区間にすると、区間の長さを`right - left`で計算でき、空の区間も`left === right`として自然に扱えます。

基本的な操作は次の二つです。

- `right`を進めて、区間を右へ広げる
- `left`を進めて、区間を左から狭める

どちらのポインタも右方向にしか動かさないことが重要です。

## なぜO(N)になるのか

二重ループのように見えても、`left`と`right`はそれぞれ最大で`N`回しか進みません。

```text
left  の移動回数: 最大N回
right の移動回数: 最大N回
合計の移動回数:  最大2N回
```

そのため、区間を広げたり狭めたりする処理が1回あたり`O(1)`であれば、全体の計算量は`O(N)`です。

`while`ループが入れ子になっていることだけを見て、必ず`O(N²)`だと判断しないようにします。計算量は、各ポインタが処理全体で何回動くかを数えて考えます。

## 適用できる条件

尺取法を使うには、区間の端を動かしたときに、条件がどのように変化するかを予測できる必要があります。

たとえば、要素がすべて`0`以上の配列では、区間の合計について次のことが成り立ちます。

- 右端を広げると、合計は減らない
- 左端を狭めると、合計は増えない

この単調性があるため、合計が上限を超えたら左端を進める、という判断ができます。

尺取法がよく使われる条件には、次のようなものがあります。

- 区間の合計が一定値以下または以上
- 区間に含まれる異なる値の個数が一定以下
- 同じ文字を含まない最長部分文字列
- 条件を満たす最短または最長の連続区間

<aside>
  <strong>重要:</strong> 条件を満たす区間が、端を動かすことで単調に変化するかを確認します。単に「連続区間の問題」というだけでは、尺取法を使えるとは限りません。
</aside>

## 例1：合計がK以下となる最長区間

すべての要素が`0`以上である配列から、合計が正の整数`K`以下となる最長の連続区間を求めます。

```text
配列: [2, 1, 3, 2, 4, 1]
K:    6

[1, 3, 2] の合計は6、長さは3
```

手順は次のとおりです。

1. `right`の要素を合計へ加えて区間を広げる
2. 合計が`K`を超えている間、`left`の要素を引いて区間を狭める
3. 条件を満たした区間の長さで答えを更新する

```ts title="合計がK以下となる最長区間"
function maxLengthAtMostK(values: number[], k: number): number {
  let left = 0;
  let right = 0;
  let sum = 0;
  let maxLength = 0;

  while (right < values.length) {
    sum += values[right];
    right += 1;

    while (left < right && sum > k) {
      sum -= values[left];
      left += 1;
    }

    maxLength = Math.max(maxLength, right - left);
  }

  return maxLength;
}

console.log(maxLengthAtMostK([2, 1, 3, 2, 4, 1], 6)); // 3
```

`while`ループを抜けた時点では、区間`[left, right)`の合計が`K`以下になっています。この「ループの前後で常に守られる条件」を **不変条件** と呼びます。

この実装では、次の不変条件を保っています。

> 答えを更新するとき、現在の区間の合計は必ず`K`以下である。

各要素は右端として1回追加され、左端から最大1回削除されるため、時間計算量は`O(N)`、追加の空間計算量は`O(1)`です。

## 例2：合計がK以上となる最短区間

次は、すべての要素が`0`以上である配列から、合計が正の整数`K`以上となる最短の連続区間を求めます。

この場合は、条件を満たしている間に左端を進めます。現在の区間が条件を満たした時点で答えの候補となり、さらに狭い区間を作れないかを試すためです。

```ts title="合計がK以上となる最短区間"
function minLengthAtLeastK(values: number[], k: number): number {
  let left = 0;
  let right = 0;
  let sum = 0;
  let minLength = Number.POSITIVE_INFINITY;

  while (right < values.length) {
    sum += values[right];
    right += 1;

    while (left < right && sum >= k) {
      minLength = Math.min(minLength, right - left);
      sum -= values[left];
      left += 1;
    }
  }

  return Number.isFinite(minLength) ? minLength : -1;
}

console.log(minLengthAtLeastK([2, 1, 3, 2, 4, 1], 7)); // 3
```

条件を満たす区間が存在しない場合は`-1`を返しています。問題によっては`0`や`null`を返すなど、要求される仕様に合わせます。

## 二つの実装を見分ける

最長区間と最短区間では、答えを更新するタイミングが異なります。

| 求めるもの              | 区間を狭める条件     | 答えを更新するタイミング   |
| ----------------------- | -------------------- | -------------------------- |
| 合計が`K`以下の最長区間 | 条件を満たさない間   | 条件を満たす状態へ戻した後 |
| 合計が`K`以上の最短区間 | 条件を満たしている間 | 区間を狭める直前           |

コードを暗記するよりも、次の二点を言葉にしてから実装すると整理しやすくなります。

1. どの状態を保ちながらポインタを動かすのか
2. どの時点の区間が答えの候補になるのか

## 区間の情報を効率よく更新する

区間が動くたびに合計や文字数を最初から数え直すと、尺取法を使っても処理が遅くなります。要素を一つ追加または削除したときの差分だけを更新します。

| 調べる条件         | 保持する情報        | 追加・削除時の処理                           |
| ------------------ | ------------------- | -------------------------------------------- |
| 区間の合計         | `sum`               | 値を加算または減算する                       |
| 異なる値の個数     | `Map`による出現回数 | 回数が`0`と`1`の間を移るとき個数を更新する   |
| 重複の有無         | `Set`または`Map`    | 右端を追加し、重複が消えるまで左端を削除する |
| 条件を満たす文字数 | 文字ごとの出現回数  | 必要数に達したかを差分で更新する             |

何を状態として保持すれば、区間の追加と削除を`O(1)`または十分小さい計算量で処理できるかを考えます。

## 負数を含む配列で使えない場合

区間和を扱う単純な尺取法では、配列の要素が`0`以上であることが重要です。負数が含まれると、右端を広げたのに合計が減ったり、左端を進めたのに合計が増えたりします。

```text
配列: [5, -10, 8]
```

`5`から区間を右へ広げて`-10`を含めると、合計は`5`から`-5`へ減ります。このため「合計が大きすぎたら左端を進める」という規則だけでは、調べるべき区間を飛ばす可能性があります。

負数を含む区間和の問題では、目的に応じて次の方法を検討します。

- 累積和
- 累積和とハッシュマップ
- 二分探索
- 単調キュー
- セグメント木

どの方法を使うかは、合計がちょうど`K`、`K`以下、`K`以上など、求める条件によって異なります。

## よくある間違い

### 区間の定義が途中で変わる

`[left, right)`と`[left, right]`を混在させると、長さの計算や要素の追加位置がずれます。実装前にどちらを使うか決めます。

この記事のコードでは、`right`を区間に含めない`[left, right)`として統一しているため、長さは`right - left`です。

### 答えを更新する位置が違う

最短区間を求めるのに、区間を条件外まで狭めてから答えを更新すると、条件を満たしていた最後の区間を失います。区間がどの状態にあるとき答えを更新するかを確認します。

### 存在しない場合を処理していない

最短値を`Infinity`で初期化した場合は、最後まで更新されなかったケースを処理する必要があります。

### 合計が数値範囲を超える

JavaScriptの`number`で整数を正確に扱える上限は`Number.MAX_SAFE_INTEGER`です。入力の最大値と配列長から合計の上限を確認し、必要であれば`bigint`を使います。

### 単調性を確認していない

サンプルで動いても、負数や特殊な条件が加わると尺取法が成立しないことがあります。「左端または右端を進めたとき、条件が一方向に変化するか」を先に確認します。

## 実装前のチェックリスト

- 対象は連続区間か
- 区間を広げる方向と狭める方向で条件が単調に変化するか
- `left`と`right`を後戻りさせずに済むか
- 区間の状態を差分で更新できるか
- 最長、最短、個数のどれを求めるのか
- 答えを更新するタイミングはどこか
- 空配列や条件を満たす区間がない場合を処理したか
- 数値の最大値は使用する型に収まるか

## まとめ

尺取法の本質は、連続区間の左右の端を一方向に動かし、過去の調査をやり直さないことです。

1. 調べる区間と保持する状態を決める
2. 右端を広げる
3. 条件に応じて左端を狭める
4. 適切なタイミングで答えを更新する

この流れが成立するかどうかは、問題の条件に単調性があるかで決まります。テンプレートをそのまま当てはめるのではなく、不変条件と答えを更新するタイミングを説明できる状態で実装することが重要です。
